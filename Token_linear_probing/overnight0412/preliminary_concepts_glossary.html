R² — 결정계수 (coefficient of determination)
회귀 모델이 정답(실제값)을 얼마나 잘 설명하는가
한 줄 정의
예측값이 실제값의 분산을 얼마나 설명하는지의 비율. 0~1 사이 값.
수식
R² = 1 − SS_res / SS_tot SS_res = Σ(y_i − ŷ_i)² ← 예측 오차의 합 SS_tot = Σ(y_i − ȳ)² ← 실제값의 전체 분산
직관적 해석
R² = 1.0
완벽
예측 = 실제
R² = 0.5
절반 설명
분산의 50%만
R² = 0.0
무의미
평균과 다를 바 없음
이 실험에서 쓰인 방식
위치 예측 (Position R²): 프레임별 mean-pooled 벡터(D-dim)로 물체의 x좌표(또는 y좌표)를 Ridge Regression으로 예측 → R² 계산.
R²(x) = 0.727 → "D-dim 벡터가 물체의 x좌표 분산 중 72.7%를 설명한다" = 위치 정보가 dim 안에 담겨있다.
R² > 0.9 → 정보가 매우 강하게 인코딩됨
R² ≈ 0 → 해당 정보가 벡터 안에 없음 (또는 선형으로 접근 불가)
F-통계량 (One-way ANOVA F-statistic)
여러 그룹 간에 평균 차이가 유의미하게 존재하는가
한 줄 정의
그룹 간 분산 / 그룹 내 분산의 비율. 높을수록 그룹이 잘 분리됨.
수식
F = MS_between / MS_within MS_between = SS_between / (k−1) ← 그룹 간 분산 (k = 그룹 수) MS_within = SS_within / (N−k) ← 그룹 내 분산 (N = 전체 샘플 수) SS_between = Σ n_j(ȳ_j − ȳ)² ← 그룹 평균 vs 전체 평균 SS_within = ΣΣ(y_ij − ȳ_j)² ← 각 샘플 vs 해당 그룹 평균
직관적 그림
dim d 값의 분포 — 4방향 그룹별
F가 낮을 때 (그룹이 섞임) — direction 정보 없음
F가 높을 때 (그룹이 분리됨) — direction 정보 있음
up down left right
이 실험에서 쓰인 방식
per-dim F-stat for direction: delta 벡터의 각 dim d에 대해 4방향 그룹(up/down/left/right) 간 one-way ANOVA 계산.

F가 높은 dim d → "이 dim의 값이 방향에 따라 뚜렷하게 달라진다" → direction 정보를 담은 dim.

상위 50개(top-50) 선정 → 모델 간 겹침 분석.
F_dir top-50 평균 595 → 843: 학습 후 direction을 담는 상위 50개 dim의 신호가 40% 강해짐.
Pearson 상관계수
두 변수가 선형적으로 얼마나 함께 움직이는가
한 줄 정의
두 변수 X, Y가 같은 방향으로 얼마나 함께 변하는지. −1 ~ +1 사이.
수식
r = Σ(x_i − x̄)(y_i − ȳ) / √[Σ(x_i−x̄)² · Σ(y_i−ȳ)²] r = 1.0 → 완전 양의 상관 r = 0.0 → 상관 없음 r = −1.0 → 완전 음의 상관
이 실험에서 쓰인 방식
Position top-dim 선정 (Exp 1.2): 각 dim d에 대해 corr(X[:,d], pos_x)와 corr(X[:,d], pos_y)를 각각 계산 후 L2 결합.

corr_pos[d] = √(corr_x[d]² + corr_y[d]²)

이 값이 높은 dim = x좌표와 y좌표 모두와 강한 선형 관계 = 위치 정보를 담은 dim.
장점: 학습 없이 순수하게 선형 관계만 측정. 빠르고 해석 직관적.
한계: 비선형 관계는 못 잡음. 여러 dim이 함께 기여하는 경우 개별 dim 평가는 제한적.
Linear Probe (선형 탐침)
모델 내부 표현에 특정 정보가 선형적으로 접근 가능한가를 테스트
한 줄 정의
모델의 hidden representation(벡터) 위에 간단한 선형 모델만 얹어서 특정 정보(레이블)를 얼마나 예측할 수 있는지 측정.
왜 선형으로 제한하나?
비선형 probe(MLP 등)는 너무 강해서 정보가 없어도 예측할 수 있음. 선형으로 제한하면 "이 정보가 선형적으로 접근 가능한 형태로 표현에 들어있는가"만 테스트. 표현(representation) 자체의 정보량을 깨끗하게 측정.
이 실험에서 사용한 두 가지 probe
Ridge Regression (회귀)
연속값 예측 (위치 x, y)
w = (X'X + αI)⁻¹X'y
α=1.0, 닫힌 형태 풀이
→ R² 보고
Logistic Regression (분류)
클래스 예측 (방향 4개, 물체 20개)
nn.Linear + Adam
lr=0.01, 300 epochs, full-batch
→ Accuracy 보고
핵심 해석 주의
Linear probe 높음 ≠ LLM이 그 정보를 잘 쓴다.
연구자가 직접 최적 선형 조합을 찾아주는 것. LLM은 attention 메커니즘으로 접근해야 하므로 probe보다 훨씬 어려운 과제.
즉 probe accuracy가 높아도 LLM QA 성능이 낮을 수 있다 — "정보는 있는데 못 꺼내는" 상황. 이게 이 논문의 핵심 병목 주장.
PCA (주성분 분석)
고차원 데이터에서 분산이 가장 큰 방향(축)을 순서대로 추출
한 줄 정의
D-dim 데이터를 분산이 큰 방향 순으로 정렬한 새 좌표계(주성분, PC)로 변환. 첫 k개 PC가 전체 분산의 대부분을 설명.
주성분(PC)이란?
원본 D-dim 벡터의 선형 조합. PC1은 분산이 가장 큰 방향, PC2는 PC1과 직교하면서 다음으로 분산이 큰 방향, … 이런 식으로 D개 PC가 만들어짐.
이 실험에서 쓰인 방식 (Exp 1.1)
pre_mean (64000, 1152) → 10000개 subsample → PCA 1000개

질문: "PC 상위 k개만으로 위치(x, y)를 예측할 수 있는가?"

결과: PC 50개 (분산 82% 설명) → R²(x) = 0.06 → 위치 정보는 상위 PC에 없다!
→ 위치 정보가 전체 dim에 분산되어 있다는 증거
분산 설명량 vs 정보량의 차이
흔한 오해: "PC 50개가 분산 82%를 설명하면 그 안에 대부분의 정보가 있다"

실제: 분산이 크다 ≠ 위치 정보가 있다. 분산이 큰 방향은 보통 semantic(물체 종류) 정보. 위치 정보는 작은 분산의 여러 dim에 분산되어 있을 수 있음.
K-fold Cross-validation
모델의 일반화 성능을 overfitting 없이 신뢰성 있게 측정
한 줄 정의
데이터를 k개 fold로 나눠 k번 반복해서 각기 다른 fold를 test set으로 사용. 결과를 평균 내면 train/test split의 운에 덜 좌우되는 안정적 평가.

최종 성능 = 5번의 test 결과 평균. 이 실험에서 std < 0.01 → 매우 안정적.
주의: Scaler는 fold마다 따로 fit
StandardScaler를 전체 데이터에 fit하면 data leakage 발생. 반드시 train fold에만 fit, test fold에 transform만 적용해야 공정한 평가.

왜 task마다 기준이 다른가?
Position은 연속값(x, y좌표) → 상관계수가 자연스러운 측정. Direction/Object는 이산 클래스 → 그룹 간 분리도인 F-통계량이 적합. 모두 학습 없이 개별 dim을 독립적으로 평가한다는 공통점이 있음.
Linear probe weight와의 차이
이 실험 방식 (per-dim univariate): 각 dim을 독립적으로 봄. "이 dim 하나만으로 target을 얼마나 예측하는가."

Linear probe weight 방식 (multivariate): 모든 dim이 동시에 협력해서 예측. 개별 dim의 기여보다 전체 조합을 최적화.

→ 둘이 꼭 같은 dim을 top으로 고르진 않는다. 이 실험은 univariate 방식.
Top-k Overlap 분석
각 task의 top-50 집합을 set으로 만들어 교집합 계산:
overlap = len(set_A ∩ set_B)

랜덤 기대값: 50 × 50 / D = 50 × 50 / 3584 ≈ 0.7개
→ Position ∩ Direction = 30~37개 (랜덤의 ~50배) → 같은 dim 공유
→ Position ∩ Object = 0~3개 (랜덤 수준) → 완전히 다른 dim
Delta vector (차분 벡터)
두 시점의 표현 차이로 "변화량(운동)"을 추출
한 줄 정의
마지막 프레임 표현에서 첫 프레임 표현을 뺀 벡터. 위치의 절대값이 아닌 위치의 변화량 = 운동 방향을 담음.
delta = feat[:, -1, :] − feat[:, 0, :] feat shape: (N, T, D) → delta shape: (N, D) 즉 N개 비디오 각각에 대해 (마지막프레임 − 첫프레임) 벡터 하나
왜 delta를 쓰는가?
단일 프레임만으로는
위치(절대값)는 알 수 있지만, 어디서 어디로 움직였는지(방향)는 프레임 하나로 알 수 없음. Direction acc ≈ 0.37~0.41
Delta 사용 시
두 프레임의 위치 변화 = 방향 정보 직접 포함. Direction acc = 0.88~0.92
직관
GPS 좌표 (37.5°N, 127.0°E) → 위치만 알고 이동 방향 모름.
(37.5°N, 127.0°E) → (37.6°N, 127.0°E) 의 차이 = (0.1, 0) → 북쪽으로 이동.

Vision token의 mean-pooled 벡터도 마찬가지: 한 프레임은 위치를, 두 프레임의 차이는 방향을 담는다.
